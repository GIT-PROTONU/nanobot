🛠 Skills · capabilities

            
loading…

            

            

              
            

            
Each capability is a self-documenting skills/*.md file —
              drop a new one in and Reload to add a skill, no code change. The brain also
              picks one on its own on a skill beat. 👍 Like a skill (repeatable) to
              make the brain perform it more often — the count weights the skill-beat lottery;
              shift-click a 👍 to take one back. Action skills (move / LED) run only when
              skills_allow_actions is enabled in robot.yaml; motion
              stays clamped by slam_nav.

            
🧪 Workshop · skills on trial

            
loading…

            

            
While in reflection mode the robot mines its experience to invent
              or improve a skill, then puts it on trial. A trial earns permanence on its own
              after enough good runs + 👍 with no errors, or you can Keep / Kill it
              here. Kill deletes the trial file; Keep adopts it for good.

Lidar (LDS)

            

              rate–
              points–
              lost/rev–
              rx errors–
            

            

              

            
/scan rate is motor-bound (~5 Hz max); the slider decimates lower.
              lost/rev = readings missing from the last revolution; rx errors =
              bad-checksum packets since start (climbing fast = degraded serial RX wiring).
              When no scans arrive at all this shows why: "no RX data" = 0 bytes on the
              UART (check the LDS→SBC wire), "RX garbled" = bytes but no valid packets.
